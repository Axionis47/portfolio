Memory is a Policy Layer

Governance, Not Storage

I used to think of agent memory as a retrieval problem. Embed, store, fetch. That turned out to be the easy part. The hard part was governance. What gets remembered, what gets expired, and what gets withheld.

I had a system where multiple agents shared a retrieval layer. One agent's working memory leaked into another agent's context. A planner started seeing raw data it was never designed for. The outputs got worse in ways that were hard to diagnose because the contamination was subtle.

Freshness was the one I ignored the longest. A fact retrieved from memory is a claim that something was true at write time. It may have been renamed, removed, or never merged. When I treated stale memory the same as current observation, I was building a hallucination machine with extra steps.

I started treating memory the same way I would treat access control in a traditional system. Expiry rules, staleness scoring, source-of-truth hierarchies. Who can read what, under what conditions, with what confidence level. These turned out to be policy decisions, not implementation details.

The framework I settled on: treat every memory write as a claim with a timestamp and a trust level. Treat every memory read as a hypothesis that needs verification against current state. Before acting on remembered information, check that it still holds. Bad memory turned out to be significantly worse than no memory at all.
