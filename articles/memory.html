The Verification Surface

For every decision, ask: how would I catch this if it is wrong?

Every system has decisions you can verify and decisions you have to trust. The engineering work is maximizing the first category and being honest about the second. If you cannot describe how you would catch a wrong output, you do not understand your system well enough to ship it.

The enterprise RAG system has the most complete verification surface I have built. Three layers of ABAC policy — index-time booleans, retrieval-time attribute checks, post-retrieval PDP filtering — each independently verifiable. The answer is either leaked or it is not. There is no gray area, no "mostly correct." The verification is binary: zero cross-role leakage across every combination of role, document, and query type.

The causal inference agent's verification is model-against-model. The critique loop takes a method selected by one agent and has another agent check whether the statistical assumptions hold against the actual dataset. Up to 3 iterations. This is probabilistic verification, not exhaustive. But it is explicit about what it checks: assumptions, not vibes.

The knowledge distillation pipeline verifies through metric fidelity. The student's nDCG@10 is measured against the teacher's on the same held-out queries. 0.882 vs. 0.910. That gap is the verification surface — it tells me exactly how much ranking quality I lost in compression. If the gap were larger, I would know where to look: the loss components, the hard-negative mining stages, the temperature schedule.

LaneGuard's conformal prediction sets are verification made visible. For each pixel prediction, the system outputs a set of possible classes with a statistical coverage guarantee. When the prediction set is large, the model is uncertain. When it contains only one class, the model is confident and calibrated. The size of the set is the verification signal.

The GNN recommendation system's ablation studies are retrospective verification. I removed FFN layers and accuracy improved. I changed attention heads and measured the delta. Each ablation answers a specific question: is this component contributing or parasitic? Without ablations, I would be shipping architecture I do not understand.

A system you cannot verify is a system you do not control.
