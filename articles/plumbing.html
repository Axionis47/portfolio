Autonomy is a Spectrum, Not a Switch

Each project sits at a different point. The skill is picking the right level.

People talk about AI systems as either "autonomous" or "not autonomous." That framing is useless. Every system I have built sits at a different point on the autonomy spectrum, and the engineering decision is choosing the minimum level of autonomy that solves the problem — then not going one step further.

At the bottom of the spectrum: the GNN recommendation system and the knowledge distillation pipeline. These are pure learned representations. No runtime decisions. No tool calls. No reasoning. The model takes an input, produces a score, and the system ranks by that score. All the intelligence is in the training loss and the architecture. Autonomy: zero. Verification: straightforward.

One level up: LaneGuard and the credit risk engine. The model makes a prediction — segment these pixels, estimate this default probability — but the system wraps it in constraints. Conformal prediction sets bound what LaneGuard is allowed to claim. SHAP decomposition and fairness monitoring bound what the credit model's outputs mean. The model has learned judgment. The system has hard limits on how that judgment is used.

Higher still: the enterprise RAG system and the OSCE grader. The model synthesizes, summarizes, and evaluates. These are generative tasks with real degrees of freedom. But the RAG system locks out access control decisions entirely. The OSCE grader locks out unsupported assertions through citation-gating. The model is free within a cage.

Near the top: the causal inference agent. Thirteen specialist agents that select methods, critique each other, and produce executable notebooks. The model makes genuine analytical decisions — which DAG algorithm, which estimator, whether the assumptions hold. But the critique loop adds a verification layer. Autonomy is high; it is checked, not unchecked.

At the top: the AI transformation agent. The model hypothesizes freely about where AI can create value across industries. It forms theories, searches for evidence, synthesizes reports. The autonomy is the highest of any system I have built. But every hypothesis is evidence-traced, and the query budget is capped at 25 to prevent runaway exploration. Even maximum autonomy has an engineering leash.

The right question is never "should the model decide?" It is "how much should the model decide, and what catches it when it is wrong?"
