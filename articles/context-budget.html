Confidence is Not Calibration

The model says it is sure. That means nothing until you verify externally.

A model outputs a number. A softmax probability, a logit, a token likelihood. People treat that number as confidence. It is not. It is a score that has no obligation to reflect reality unless you force it to.

LaneGuard taught me this the hard way. The SegFormer would output pixel-level segmentation masks with high softmax scores — 0.95, 0.98 — on regions that were completely wrong. A shadow on the road, a faded lane marking. The model was confidently incorrect. So I wrapped it in conformal prediction: instead of asking "how sure are you?", I asked "given a calibration set, what prediction set guarantees coverage at this error rate?" The model's own confidence became irrelevant. The statistical guarantee replaced it.

The credit risk engine has the same problem at a different scale. A PD model that outputs 0.12 probability of default is useless for Basel III if that 0.12 does not actually mean 12 out of 100 similar borrowers will default. Calibration is a regulatory requirement, not a nice-to-have. The ensemble's outputs get calibrated against observed default rates before they touch any ECL calculation.

The GNN recommendation system sidesteps this entirely. I never ask the model how confident it is about a recommendation. I measure Recall@10 on a held-out set — did the user actually interact with what the model ranked highly? The model's internal scores are just a ranking mechanism. The evaluation metric is the only confidence that matters.

In the knowledge distillation pipeline, the student model's scores had to track the teacher's soft distributions via KL divergence. The student could not just be accurate — its uncertainty had to match the teacher's uncertainty. A student that is right with wrong confidence is a broken distillation.

Never ask a model how sure it is. Measure it yourself.
