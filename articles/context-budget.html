Context is a Budget

Minimum Sufficient Context

Every token in the context window either helps the decision or hurts it. There is no neutral information. Extra context doesn't make the model "more informed" — it makes it more confused.

Anthropic calls it the "smallest set of high-signal tokens that maximize the desired outcome." I call it a budget. You have a finite number of slots, and every slot you waste on irrelevant information is a slot that could have held something useful.

A router needs the query and maybe a metadata snippet. Not the chat history. Not all retrieved documents. A summarizer needs one text chunk and its instructions. Not the user's profile. Not unrelated state.

The test is simple: for each piece of context, ask "would removing this change the output?" If the answer is no, it shouldn't be there. If you're adding context "just in case," you've masked an unclear task or a missing step.

Context rot is real. As windows get larger, LLMs lose focus, confuse facts, or latch onto irrelevant details. The solution isn't bigger windows. It's discipline about what enters them.

Frame each component's role as a single decision boundary. Ask: "what information would actually change the output of this step?" Everything else is noise. If you find yourself adding context just in case, pause. That usually masks an unclear task or a missing step.

Selective retrieval beats bulk loading. Instead of dumping all data upfront, maintain an index and fetch relevant pieces on demand — just-in-time context. Keep file paths in context, load file contents via tools. This mimics how humans search their environment: on demand, not preloaded.

Larger windows don't solve context rot. Discipline does.
