Semantic Search · Knowledge Distillation

Train a small model to think like a big one. 97% accuracy. 100x faster. $4.

A search system returns the right answer in 100 milliseconds. The user has already moved on. The large model knows the answer — it’s just too slow to matter. So we taught a model 17x smaller to think like it. The student reaches 97% of the teacher’s accuracy at a fraction of the cost — $4 to train, 1ms to answer.

The hard part

Without hard negative mining, the student plateaus at 85% of the teacher. The breakthrough was a 3-stage curriculum: start with cheap BM25 keyword negatives, then let the teacher score and filter them (keeping only high-confidence disagreements), then let the student mine its own adversarial negatives every 500 steps. Each stage forces the student to learn harder distinctions. That’s what got us from 85% to 97%.

Query Processing

Raw text is prefixed with "query: " to match E5’s training format. This prefix tells the encoder to produce a query-optimized embedding rather than a passage embedding.

Student Encoder — intfloat/e5-small-v2

6-layer BERT bi-encoder. Encodes query independently from documents (unlike the teacher which needs both together). This is what makes it fast — documents are pre-encoded once and indexed.

Architecture: BERT 6-layer · 33M params · 384-dim output
Pooling: Mean pooling over token embeddings
Latency: ~1ms per query on CPU
FAISS HNSW Index

Hierarchical Navigable Small World graph for approximate nearest neighbor search. Trades a tiny amount of recall for massive speed gains over brute-force.

M = 32 connections per node
efConstruction = 200 (build quality)
efSearch = 50 (query quality)
Recall: >97% · Latency: ~10ms for 10K docs
FastAPI Serving Layer

Production API with security middleware stack: rate limiting (100 req/min), API key auth (SHA-256 hashed), HSTS/CSP headers, CORS, health checks for Kubernetes readiness/liveness probes.

How it learns — Training Pipeline
01
Data — MS MARCO
603 queries and 643 passages from Microsoft’s passage ranking dataset. Chunked to 512 tokens with 80-token stride for overlapping context windows.
02
Hard Negative Mining (3 stages)
Documents that look relevant but aren’t. This is where the student learns to make fine distinctions.
Stage A: BM25 keyword negatives (fast, cheap)
Stage B: Teacher-scored semantic negatives (>0.7 confidence filter)
Stage C: ANCE adversarial — student mines its own negatives every 500 steps
03
Knowledge Distillation
Teacher scores every query-document pair. Student learns to reproduce those scores through three combined loss functions.
60% Margin-MSE — learn score differences between documents
20% Listwise KD — learn the full ranking distribution (KL divergence)
20% Contrastive — learn query-doc similarity via in-batch negatives
04
Temperature Annealing
Teacher’s score distribution starts soft (T=4.0) so the student learns coarse ranking, then sharpens (T=2.0) for fine distinctions. Linear decay over 3 epochs.
05
Export & Index
Trained student encodes all documents into 384-dim vectors. FAISS HNSW index is built. Model + index are packaged for Docker deployment on Cloud Run.
Training: 8.5 hours on GCP CPU (~$4)
Index: HNSW M=32 · Docker multi-stage build · Cloud Run deploy
